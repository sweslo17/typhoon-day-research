# Typhoon Day Research Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a reproducible research package for Taiwan typhoon-day decisions, an interpretable probability method, and Medium-ready statistical science drafts.

**Architecture:** A small Python package owns parsing, feature engineering, analysis, modeling, and article generation. Raw source artifacts are kept as local files; derived CSV/Markdown/SVG/JSON outputs are regenerated by CLI commands. Network fetchers are separated from parsers so tests can run offline.

**Tech Stack:** Python standard library, optional Poetry project metadata, local CSV/HTML/JSON files, loguru-compatible logging boundary if external dependencies are added later.

## Global Constraints

- Work order is A, C, B: research data package first, prediction model second, Medium series drafts third.
- Election timing must include presidential elections and local elections.
- Election features must distinguish same-year elections from upcoming elections in the following year.
- Use official sources where available: DGPA historical stop-work/class notices, CWA CODiS observations, CWA typhoon wind/rain forecast products, national legal standards.
- Keep outputs reproducible from local source files and deterministic code.
- Do not require external services or databases for baseline runs.
